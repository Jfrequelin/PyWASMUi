# Security Policy

## Supported Versions

Security updates are provided for the latest stable version on `main`.

## Reporting a Vulnerability

Please do not open a public issue for suspected vulnerabilities.

1. Open a private security advisory on GitHub, or
2. Contact the maintainers directly if private advisory is not available.

Include as much detail as possible:

- affected version/commit,
- reproduction steps,
- impact assessment,
- possible mitigation.

We will acknowledge receipt as quickly as possible and keep you informed during
triage and remediation.

## Disclosure

We follow coordinated disclosure. Once a fix is available, we may publish:

- impacted versions,
- mitigation guidance,
- credit (if requested).
